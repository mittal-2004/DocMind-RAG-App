## 🔍 Overview

**DocMind** is a production-deployed, AI-powered document Q&A system built on **Retrieval-Augmented Generation (RAG)**. Users upload a PDF document and ask natural language questions — the system retrieves the most relevant content using **semantic vector search** and generates precise, context-grounded answers via **Mistral AI**.

Unlike simple chatbots, DocMind's RAG architecture ensures answers are always **anchored to your document** — significantly reducing hallucinations and improving factual accuracy.

**Key Highlights:**
- 🚀 Fully deployed and publicly accessible on Streamlit Cloud
- 🧠 End-to-end RAG pipeline built from scratch using LangChain
- 🔍 Semantic search via ChromaDB + Sentence Transformers
- 🤖 Mistral AI for high-quality, context-aware answer generation
- ⚡ Real-time interactive UI with Streamlit

## ⚙️ RAG Pipeline — How It Works

### Phase 1: Document Ingestion (on PDF Upload)

```
PDF File
  → Parse raw text           (PyPDF / Unstructured)
  → Split into chunks        (LangChain RecursiveCharacterTextSplitter)
  → Generate embeddings      (Sentence Transformers)
  → Store vectors in DB      (ChromaDB)
```

### Phase 2: Question Answering (on User Query)

```
User Question
  → Embed the question       (Sentence Transformers)
  → Similarity search        (ChromaDB cosine similarity)
  → Retrieve Top-K chunks    (most relevant document sections)
  → Build LLM prompt         ([System] + [Context] + [Question])
  → Generate answer          (Mistral AI)
  → Display to user          (Streamlit UI)
```

### Why RAG over plain LLM?

| Plain LLM | RAG (DocMind) |
|---|---|
| Hallucinates — makes up answers | Answers grounded in your actual document |
| No access to your private PDF | Directly references your uploaded content |
| Generic, vague responses | Precise, document-specific answers |
| Limited by context window size | Handles large documents via smart chunking |

## 🚀 Getting Started Locally

### Prerequisites

- Python 3.11+
- Mistral AI API Key → [Get it here](https://console.mistral.ai/)

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/mittal-2004/DocMind-RAG-App.git
cd DocMind-RAG-App

# 2. Create and activate virtual environment
python -m venv venv
source venv/bin/activate          # Mac / Linux
venv\Scripts\activate             # Windows

# 3. Install dependencies
pip install -r requirements.txt

# 4. Create .env file with your API key
echo "MISTRAL_API_KEY=your_mistral_api_key_here" > .env

# 5. Run the app
streamlit run app.py
```

Open your browser at: `http://localhost:8501`

## 🔐 Environment Variables

**Local development** — create a `.env` file in the project root:

```env
MISTRAL_API_KEY=your_mistral_api_key_here
```

**Streamlit Cloud deployment** — navigate to:

`App Dashboard → Manage App → Settings → Secrets`

```toml
MISTRAL_API_KEY = "your_mistral_api_key_here"
```

> ⚠️ Never hardcode API keys in source code or commit `.env` to GitHub.

## ☁️ Deployment

This app is deployed on **Streamlit Cloud** (free tier).

**To deploy your own fork:**

1. Fork this repository on GitHub
2. Go to [share.streamlit.io](https://share.streamlit.io)
3. Connect your GitHub account and select this repo
4. Set `app.py` as the main module
5. Add `MISTRAL_API_KEY` under App Secrets
6. Click **Deploy** 🚀
